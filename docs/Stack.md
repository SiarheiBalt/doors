🧱 Архитектура проекта на одном белорусском VPS

1. Фронтенд: Полностью статический Next.js (output: 'export'). Папка out просто копируется на сервер. В next.config.js включается images: { unoptimized: true }.
2. Бэкенд: API на Node.js, запущенный на сервере локально на свободном порту (например, 5000). Он обрабатывает корзину, заказы и админку.
3. База данных: Локальная PostgreSQL. Prisma подключается к ней внутри сервера по адресу localhost:5432. (Файловая SQLite не подходит для магазина из-за блокировок базы).
4. Картинки дверей: Менеджер загружает их через админку \(\rightarrow \) Node.js бэкенд сжимает их через библиотеку sharp в формат WebP \(\rightarrow \) файлы сохраняются в папку на диске сервера.
5. Nginx (Главный диспетчер): Принимает весь внешний трафик. Запросы к сайту отдает как статику Next.js, запросы /api/ перенаправляет на Node.js, а запросы к /uploads/ мгновенно отдает как картинки. Здесь же бесплатно подключается SSL от Let’s Encrypt.

Пошаговый план запуска бизнесаШаг 
1. Разработка: Написать код статического фронтенда Next.js и API бэкенда на Node.js с Prisma. 
2. Хостинг: Купить VPS у белорусского провайдера (hoster.by, A1, ActiveCloud) и купить домен .by. 
3. Легализация сайта: При покупке хостинга поставить галочку «Регистрация в БелГИЭ» (хостер сделает всё сам за ~20 BYN) [🛠️]. 
4. Деплой: Установить на VPS PostgreSQL и Nginx, настроить маршруты трафика и загрузить файлы проекта. 
5. Деньги: Заключить договор с белорусским агрегатором (bePaid, Webpay) и подключить их готовое API/виджет для приема карт и ЕРИП. 
6. Налоги и Закон: Подключить программную кассу (iKassa, Электронный чек) для чеков в налоговую. Подать заявление в Гродненский исполком для включения магазина в Торговый реестр РБ.
